## 三、命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 实体 | PascalCase 单数 | `Agent`, `Skill`, `User` |
| 数据表 | snake_case 复数 | `agents`, `skills`, `users` |
| 关联表 | `{a}_{b}s` | `agent_skills`, `agent_mcps` |
| API 路径 | kebab-case 复数 | `/api/v1/agents` |
| DTO 字段 | snake_case | `user_id`, `workspace_name` |
| Go 字段 | PascalCase + JSON tag | `UserID string \`json:"user_id"\`` |

## 五、禁用词 / 歧义词

以下词容易引起歧义，请**避免使用**：

- ❌ `data`（太泛，说清是什么数据）
- ❌ `info`（同上）
- ❌ `obj`（用具体类型名）
- ❌ `temp` / `tmp`（说明是临时什么）

---

## 六、规则

1. **一个概念一个词**：同一概念在所有地方用同一词（如"用户"不要和"使用者"/"账号"混用）
2. **DB 列名与代码字段对应**：`user_id` ↔ `UserID` / `userId`
3. **新增术语先登记**：AI 发现新术语时，提醒人类更新本文件
