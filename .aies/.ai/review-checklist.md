# AI 代码审查清单（通用版）

> ⚠️ 每次 AI 生成代码后，**必须**逐项检查以下清单。
> AI 写的代码看起来完美，但往往隐藏逻辑盲区。
>
> **本清单是通用模板，每个项目应根据自身语言/框架补充具体检查项。**

---

## 一、架构合规检查

- [ ] **分层是否正确**：各层职责清晰，未越权？
- [ ] **依赖方向是否正确**：是否存在逆向依赖？
- [ ] **文件放置是否正确**：新文件在正确目录下？
- [ ] **命名是否一致**：类/方法/文件名遵循项目命名约定？
- [ ] **Context/Session 全链路透传**：请求上下文是否从入口传到数据层？

---

## 二、数据安全检查

- [ ] **SQL 注入**：使用了参数化查询？无拼接 SQL？
- [ ] **越权访问**：接口校验了资源归属？（如用户只能操作自己的数据）
- [ ] **敏感信息泄漏**：错误信息未暴露内部实现细节（表名、SQL、路径）？
- [ ] **输入校验**：对所有用户输入做了长度/格式/范围校验？
- [ ] **密钥/凭证**：未硬编码在代码中？使用配置或环境变量？

---

## 三、数据完整性检查

- [ ] **事务完整性**：多表/多步操作使用了事务？异常路径正确回滚？
- [ ] **级联操作**：删除主记录时清理了关联数据？
- [ ] **唯一约束**：需要唯一的字段加了 unique index？
- [ ] **外键一致性**：关联 ID 指向的记录存在？
- [ ] **空值处理**：NULL vs 空字符串 vs 零值的业务含义清晰？

---

## 四、边界条件检查

- [ ] **空值处理**：传入 null/undefined/空数组时正常处理？
- [ ] **超大输入**：超长字符串、超大分页 size 有限制？
- [ ] **重复操作**：接口幂等？重复创建/删除安全？
- [ ] **并发安全**：有竞态条件？（如同时创建同名资源）
- [ ] **零值陷阱**：Update 请求的指针字段正确处理"设为零值"与"不更新"？

---

## 五、错误处理检查

- [ ] **错误分类正确**：4xx / 5xx 使用合理？
- [ ] **错误信息有意义**：用户能从错误信息中知道如何修正？
- [ ] **错误不被忽略**：是否有 `_ = someFunc()` 忽略了不该忽略的错误？
- [ ] **错误不被吞掉**：内层错误正确传递到外层？
- [ ] **错误码统一**：使用项目统一错误码定义，未硬编码字符串？

---

## 六、性能检查

- [ ] **N+1 查询**：列表查询中循环查数据库？使用了预加载？
- [ ] **不必要的预加载**：列表接口加载了不需要的关联数据？
- [ ] **分页查询**：大数据量使用了分页？限制了最大 page size？
- [ ] **索引覆盖**：查询条件中的字段有数据库索引？
- [ ] **缓存策略**：热点数据有缓存？缓存失效策略合理？

---

## 七、日志与可观测性

- [ ] **关键操作有日志**：创建/更新/删除记录了操作日志？
- [ ] **错误有日志**：业务层和数据层的错误记录了日志？
- [ ] **日志不含敏感信息**：日志避免了密码、token、身份证等敏感字段？
- [ ] **日志级别合理**：debug/info/warn/error 使用正确？
- [ ] **trace/request id**：能够追踪单次请求的完整调用链？

---

## 八、AI 常见盲区（重点检查）

- [ ] **硬编码值**：URL、端口、密钥、魔法数字未硬编码？
- [ ] **遗漏默认值**：Create 时设置了所有必要的默认值？
- [ ] **JSON tag 遗漏**：新增字段都有 JSON tag / serializer 注解？
- [ ] **时区问题**：时间字段统一时区处理？
- [ ] **整数溢出**：大数字段类型够用（bigint vs int）？
- [ ] **编码问题**：字符串编码统一 UTF-8？
- [ ] **索引/文档更新**：`.ai/index.md` 和 API 文档需要更新？
- [ ] **单位统一**：时间（ms/s）、大小（B/KB/MB）单位一致？

---

## 九、测试覆盖（可选）

- [ ] **核心路径有测试**：主要业务逻辑有单元测试？
- [ ] **边界用例**：空值、错误输入、超大输入有测试？
- [ ] **测试隔离**：测试不依赖外部环境（DB、网络）？或使用 mock？

---

## 使用方式

1. AI 生成代码后，对照此清单逐项检查
2. 发现问题标注 `❌`，并要求 AI 修正
3. 全部通过后标注 `✅`，可以提交
4. 如果发现新的盲区，**补充到此清单中**（并在 `.ai/changelog.md` 记录）
